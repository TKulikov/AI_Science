# Knowledge Graph RAG — база знаний горно-металлургической отрасли

Система строит граф знаний и векторный индекс из корпуса документов
(статьи, отчёты, патенты, справочники) и отвечает на вопросы вида
«что делали с материалом X при режиме Y и какой эффект на свойство Z»
со ссылками на источники.

## Стек

- **LLM и эмбеддинги** — YandexGPT через Yandex AI Studio SDK
- **Граф и поиск** — LlamaIndex (PropertyGraphIndex + VectorStoreIndex)
- **API** — FastAPI

## Структура

```
data/raw/                 корпус документов (кладёте сюда)
data/graph_store/         граф (создаётся при индексации)
data/vector_store/        векторный индекс (создаётся при индексации)
src/processing/loader.py  загрузка документов батчами
src/graph/store.py        построение индексов
src/query/query_engine.py  поиск граф и фолбэк вектор
src/index.py          индексация
src/processing/extract_archives.py    распаковка zip/rar
main.py                   веб-сервер
index.html                интерфейс
```

## Установка

```bash
pip install -r requirements.txt
```

Для `.rar` дополнительно нужен системный `unrar` или 7-Zip
(Windows: положить `unrar.exe` в PATH; Linux: `apt install unrar`).

## Настройка ключей

Скопируйте `.env.example` в `.env` и впишите данные Yandex Cloud:

```
FOLDER_ID=ваш_folder_id
API_KEY=ваш_api_key
```

У сервисного аккаунта должна быть роль `ai.languageModels.user`,
иначе вызовы вернут `PERMISSION_DENIED`.

## Порядок запуска, если графовой БД ещё нет

Если папок `data/graph_store` и `data/vector_store` нет — сначала
строим индексы, потом запускаем сервер.

**1. Положить корпус.** Документы в `data/raw` (можно вложенными
папками — год и журнал берутся из структуры пути, например
`data/raw/Журналы/Цветные металлы/2024/`).

**2. Распаковать архивы** (если в корпусе есть zip/rar):

```bash
python scripts/extract_archives.py --data-dir data/raw
```

**3. Построить индексы:**

```bash
python scripts/index.py --data-dir data/raw
```

Идёт потоково, батчами — корпус не держится в памяти целиком.
После каждого батча прогресс сохраняется на диск, поэтому при
обрыве не нужно начинать заново. По окончании появятся
`data/graph_store` и `data/vector_store`.

Индексация обращается к YandexGPT на каждый фрагмент, поэтому на
большом корпусе занимает время. Сначала прогоните на 20-30 файлах,
замерьте, потом на всём объёме.

**5. Запустить сервер:**

```bash
python main.py
```

Открыть http://127.0.0.1:8000 и задавать вопросы.

---

## Порядок запуска, если индексы уже построены

Папки `data/graph_store` и `data/vector_store` на месте — шаги 1-4
пропускаем, сразу:

```bash
python main.py
```

Индексы загружаются один раз при старте сервера.

## Настройки (.env)

| Переменная | Смысл | По умолчанию |
|---|---|---|
| `LOAD_BATCH_SIZE` | документов в батче при индексации | 50 |
| `MAX_FILE_MB` | файлы крупнее пропускаются | 60 |
| `SCAN_MIN_CHARS_PER_PAGE` | ниже порога PDF считается сканом | 50 |
| `KG_NUM_WORKERS` | параллельные вызовы Яндекса | 8 |
| `CHUNK_SIZE` | размер фрагмента при нарезке | 500 |

При нехватке памяти уменьшайте `LOAD_BATCH_SIZE`.
При ошибках rate limit от Яндекса — снижайте `KG_NUM_WORKERS`.

## Как работает поиск

Запрос парсится (сущности, числовые ограничения вида «≤300 мг/л»,
год, язык), затем идёт в граф. Если граф дал достаточно результатов —
ответ синтезируется из него. Если нет — фолбэк на векторный
семантический поиск. Если ни там, ни там ничего не нашлось —
система сообщает, что данных по такой комбинации в корпусе нет.
